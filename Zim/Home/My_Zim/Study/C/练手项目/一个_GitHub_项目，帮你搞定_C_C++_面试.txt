好了，转入正题吧，咱们今天是来聊 C/C++ 技术面试的，二次元的东西先放一边。

长期以来，GitHub 上的 Markdown 项目一直被不少开发者所诟病，通常都会被认为是骗 Star，刷眼球的项目。但其实，作为资源分享类的仓库，里面的不少内容也实实在在帮助到了一些初入编程的开发者或应届生，减少了他们检索和汇总信息的成本。

作者开源的 interview 这个项目，其实是一份 C/C++ 技术面试基础与知识总结，里面包括语言、程序库、数据结构、算法、系统、网络、链接装载库等知识及面试经验、招聘、内推等信息。

文中的不少知识点，也均有配上代码示例和详细解说，以帮助读者更好的吸收。

不过，对一名合格的工程师而言，平时多看 GitHub 上的一些优质项目源码，给知名项目提交 PR，或在业余时间独立开发一款产品，这才是最大化提升自己编程能力的最佳途径。
