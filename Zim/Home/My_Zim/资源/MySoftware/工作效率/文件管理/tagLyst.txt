Windows 一直缺失的标签和资料库，由 tagLyst Next 来补全
01月05日
沨沄极客 沨沄极客
在文件管理方面，提到 Windows 与 macOS 之间的区别最大的一点恐怕就是标签了。标签这一功能可以跨越文件夹的限制，给一个文件打上多个标签，来实现对文件的多维管理。解决了传统的文件夹形式中，具有多种属性的文件归类问题。

严格的讲，Windows 一直以来都只有文件夹这一种组织形式，自 Windows 7 开始推出的「库」功能只能算是多个文件夹的集合，并没有解决根本问题。

Windows 的库，macOS 的标签
Windows 的库，macOS 的标签
为什么 Windows 一直没有好用的标签软件？
Windows 没有标签软件，其实并不能怪软件开发商不努力。

根本原因在于 Windows 的 文件体系中压根没有好好设计标签这个属性（你可以开启，但是并没有地方设置）。

甚至根本找不到标签这个属性
甚至根本找不到标签这个属性
我曾经做过很多尝试：使用系统级插件、第三方文件管理器，最后的效果都不理想，甚至还不如用纯文件夹系统……

究其原因，为什么 macOS 的标签好用？因为 macOS 的标签提供了系统级的支持，每个文件都能打标签。macOS 的系统、软件、应用数据相对封闭，在桌面和文档目录中几乎都是个人文件。

而 Windows 不一样，没有统一的标签属性，文件没有统一的归属地，系统默认目录和普通文件一样放在 C 盘，没有区分度，系统和软件目录都是普通的文件夹，特定软件还会产生许多缓存文件、下载文件等。让你为现有的文件都打上标签，实在是一件令人为难的事情。这些问题在 Windows 上至今都没有一个通用的方案，再加上一个标签插件，混乱程度怕是要呈几何级上升。

那么怎么才能在 Windows 里实现标签？
如果想让文件具备标签属性，其实有 3 条路可以走。

路线一：系统级支持、插件支持
Windows 作为闭源系统，标签属性的缺失，使得开发系统级的插件难度很高。如果没有微软官方的支持，做一个和 macOS 一样的系统级标签软件几乎是不可能的。

微软似乎丝毫没有增加属性的想法。这条路很难实现。

路线二：在软件中增设标签属性
在软件中增设独立的 标签 属性。

这一条路是目前市面上大多数文件管理器软件的解决方法。思路是「既然你没有，那我就自己造一个标签功能」。比如 Directory Opus 和 XYplorer ，它们就采取了这个方案，内置了软件独占的标签功能。

Directory Opus 的标记
Directory Opus 的标记
然而这类独占标签方案很难做大的原因有两点：

依赖性强：独占标签体系严重依赖于软件，不能导入导出等。标签基于文件，文件又是独立的个体。一般软件最多只能把标签方案导入导出，没办法把「某个文件带有什么标签」一起导出。这意味着一旦卸载了软件，文件的标签属性就会丢失。
通用性低：采用独立标签方案，意味着无法和其他效率软件配合使用。比如你用 Directory Opus 给文件加了一个标记，之后想用 Everything 搜索这个标记，这种操作是做不到的，你只能在软件内部使用这个标签。这就进一步降低了标签的实用性。
所以，我心目中理想的标签软件，应该是不依靠第三方软件也能独立运转，并具有通用性的软件。

只剩下了第三条路。

路线三：借助现有的文件属性实现标签
这是一条折中路线，也是一种更通用的选择：不借助第三方工具，直接用现有的文件属性实现标签。

比如利用「文件名」「文件备注」等现有的属性，来实现标签系统。 tagLyst 就采取了这种另辟蹊径的做法：将文件名的一部分作为标签。 这就来看看 tagLyst Next 是怎么解决这个难题的。

文件名后缀， 解决大难题
在我刚刚接触这款软件时（大约一年前），tagLyst 还算是一个新生软件，很多地方还不完善，我当初在 tagLyst 里添加文件之后，发现它的标签会对文件名产生影响，这会影响到我现有的文件体系，所以我放弃了它。

当初放弃 tagLyst 的原因其实只有一个——「标签」并不是作为属性附加在文件上，而是对文件名进行了修改。具体一点，就是在文件名的最后加上了 #标签 的字样。

文件后缀就是 tagLyst 的标签
文件后缀就是 tagLyst 的标签
大家刚开始上手时应该会跟我有一样的想法：「这怎么行，文件名都被这软件弄得一团糟了」。其实这恰恰解决了上面提到的依赖性强和通用性低的问题。

再好用的标签，也是附着于文件的一种属性。tagLyst 做的事情表面上只是给文件加了个后缀，但在实际体验中，这小小的后缀用处不少。

当你把文件挪到其他位置时，标签直接跟随文件名移动。即使放在 U 盘里，换了台没有安装 tagLyst 的电脑，你的标签依然不会消失，直接附着在文件名上，也不再需要导入导出这类操作了。这是对软件的低依赖性。
当你用其他软件进行搜索时，由于标签直接写在文件名上了。只要搜索 # 就可以很快搜索到需要的文件，这是高通用性。
所以，tagLyst 算是一举解决了前面的问题。相比之下，文件名的改变实在是微不足道的缺点。回过头来想想，这对于原先不擅长命名的用户，可能反而算是福音了（笑）。

用 tagLyst Next 建立一个局部标签系统
面对一个全新的软件，让你马上给所有文件都打上标签，实在是一件令人为难的事情。工作量大不说， 还容易误操作，得不偿失。

但如果换一个角度，让你把有用的文件转换为资料库，再给这些文件打上标签，这件事就变得容易接受得多，这就是 tagLyst Next 的思路。

文件和标签，一目了然
文件和标签，一目了然
tagLyst Next 并不想做一个全面标签系统，而是做一个局部标签系统。换句话说，tagLyst Next的目标不再是将所有的文件都标签化，而是让用户对有需要的文件打上标签。

这就变成了一个不错的方案。

标签只是功能点，资料库才是 tagLyst Next 的本体
我最初想法是给现有文件打上标签，而不破坏原有的目录结构，所以用起来感觉很别扭。但现在 tagLyst Next 的主打角度发生了变化——由「标签管理软件」变成了「个人资料库」。

这样一来，我的使用思路也随之发生改变，我不再把所有文件一股脑丢进去让它管理，而是把有用的文件收集起来，单独作为资料库使用。标签就成为了一个辅助检索的功能点。

支持传统的文件管理形式
支持传统的文件管理形式
经过了几个版本的更新，新版 tagLyst Next 围绕着资料库设计的功能得到了完善，不仅有好用的标签系统、常规的文件名搜索，还有完善的多维度筛选等功能。这款软件已经足以承担资料库的任务了。

方便地创建资料库
即便你对标签功能不感兴趣，tagLyst Next 也能单纯作为资料库，替你整理好资料文件。

创建方法很简单，只需把有用的文件拖入 tagLyst Next 中，不必再管多余的事，就能有个便于多方位检索的资料库了。

启用默认资料库
启用默认资料库
如果初次上手还不太熟悉，也不想影响当前的文件体系。可以在偏好设置里打开「我的下载」「我的图片」「我的音乐」「我的视频」的默认资料库功能，tagLyst Next 就会直接调用 Windows 中的默认文件夹作为资料库。你可以对这些目录直接进行筛选、检索。

美观易用的标签功能
tagLyst Next 允许用户自定义标签的分类、标签名称、标签颜色、图标样式，自定义程度很高。在标签管理界面，你可以对标签进行任意拖拽，分配到不同的分组。

自定义标签
自定义标签
在具体使用标签时，需要用户先设置一些标签。 在完成标签设置之后，就可以为文件可以打上多个类型的标签，可以很快的为文件建立不同的自定义分组、标签颜色、分配众多图标，按你需要的方式用标签组织起来。

打上标签时，只有文档文件才会被修改名称。当你选择给一个文件夹打标签的同时，可以选择为其中的子文件夹一次性全部打上相同的标签。tagLyst Next 现在允许给文件夹批量修改标签，文件夹的文件名不会被修改。

多维度筛选功能
tagLyst Next 界面的最左侧是不同的资料库，你可以把工作和个人文件分成两个资料库来使用。第二栏就是标签、目录、类型、日期四个维度的筛选工具。筛选器分为精简、标准、高级三个级别，在深度和广度上都能找到适用的筛选视图。

多维度筛选
多维度筛选
标签筛选工具有精确匹配、默认匹配、广泛匹配三种模式，允许反向选择「不含标签」，以便给尚未标记的文件打上标签。

目录筛选和系统自带的目录类似，不必多提。

类型筛选可以快速把主要的文件类型筛选出来，办公文件（.docx .PDF）和文本记事（.txt .md）是我最常用的类型。

按类型筛选
按类型筛选
日期筛选允许你对指定日期内的文件进行筛选。不过多数情况下并不需要填写精确的日期，点一下「昨天」或是「上周」，通常就能很快找到需要的文件了，一页就能显示完整。

按日期筛选
按日期筛选
要知道在默认的文件夹系统中，通常只能靠文件名进行搜索，然后肉眼查找需要的文件。tagLyst Next 的这几个筛选功能对效率的提升有着极大的帮助。

弥补原生文件管理器的不足
除了标签、筛选这种效率方面的改进，tagLyst Next 在其他方面也下了功夫。

自动探寻下级文件夹功能也深受我的喜爱，有时候文件层级很多，找一个文件需要点击多次文件夹。 如果找错了目录还需要返回上一级。 在 tagLyst Next 中就没有这个问题。 它可以把目录全部拍平，在一个页面中展示目录下的所有文件夹。

tagLyst Next 中提供了文档格式预览、灵感流瀑布式布局等视图，可以对标签的文件数量进行统计、生成标签云。

灵感流视图
上面介绍的这些体验，都是建立在使用 tagLyst Next 软件的基础上，脱离软件后你仍然可以通过搜索 #标签 的方式访问这些标签。

另外，在 tagLyst 先前版本中的一些问题，开发者都积极的完成了改进。简化标签引擎，提高检索效率，降低系统开销，各方面性能都得到了不小的提升。原先体验过的用户也可以重新下载尝试一番。

小结
在我们寻找和对比工具时，也应当了解自己的需求所在。当我多番寻找适用于 Windows 整个系统的标签软件无果之后，我发现最需要的并不一定是标签这一个小功能点，而是可以提供标签功能的工具来帮助我管理文件。tagLyst Next 就是一个相当不错的选择，不仅解决了标签的矛盾，找对了做资料库的方向，给 Windows 的文件管理提供了一个全新的思路。
