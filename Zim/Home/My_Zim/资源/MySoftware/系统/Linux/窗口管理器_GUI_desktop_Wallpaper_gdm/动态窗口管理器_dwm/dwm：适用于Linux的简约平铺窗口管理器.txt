====== dwm：适用于Linux的简约平铺窗口管理器 ======
Created 星期二 18 二月 2020

https://www.maketecheasier.com/dwm-tiling-window-manager/
   
平铺窗口管理器比它们更受欢迎的表兄弟（例如Gnome，KDE，XFCE或Fluxbox）具有多个优势。这篇文章的功能dwm将这些优势发挥到了极致。

尽管大多数平铺管理器都努力做到轻量化，但dwm却使自己陷入了2000行或更少代码行的饥肠diet。它的所有配置都在编译时完成，因此它不会读取运行时配置文件。它使用标签（数字1到9）而不是任意命名的窗口空间来将程序组合在一起。它也可以完全用键盘命令运行，尽管它确实包含鼠标支持，以便在适当时选择和拖动窗口。

如果您正在寻找简捷的体验或对当前桌面环境的阻碍，dwm可以提供完全一样的体验。在尝试某些功能之前，让我们开始安装。

安装
使用git来获取源代码很简单：

git clone https：// git.suckless.org / dwm
在这里，您可以编辑dwm的“ config.mk”文件来更改其安装目录（默认情况下为“ / usr / local /”），并可以通过其“ config.h”文件来更改键绑定。然后，您可以使用安装make clean install。

一些Linux发行版提供了预编译的二进制文件，例如Ubuntu和Arch。对于Ubuntu，请安装：

sudo  apt-get install dwm
对于Arch，您需要访问其用户存储库并使用进行构建/安装makepkg。

但是请注意，预编译的二进制文件通常会附加到各自的程序管理器上，例如Ubuntu的APT和Arch的Pacman，它们会在dwm更新时安装“ config.mk”和“ config.h”的新副本。因此，所有更新都将写入您对这些配置文件所做的任何更改。Arch建议您制作自己的软件包以保存更改。
Dwm除X库外还具有一些依赖性。如果您从源代码安装了dwm，那么您将想要抓住dmenu（一种菜单工具）和st（一种终端）。如果您使用的是程序包管理器，它应该已经选择了那些额外的程序包。

开始dwm
安装后，如果您使用的是Ubuntu，则可以从登录屏幕的菜单中选择“ dwm”。对于其他窗口管理器，可能更容易编辑X配置文件“ .xinitrc”。

该配置文件告诉X显示服务器使用哪个窗口管理器。您可以先在主文件夹中找到该文件，然后再对其进行编辑。


打开“ /home/username/.xinitrc”并滚动到底部以查找“ exec gnome-session”之类的条目。启动X时，必须将找到的所有“ exec…”替换为“ exec dwm”以启动dwm。在此示例中，更改如下所示：

#exec gnome-session 
exec dwm
.xinitrc中任何文本前面的“＃”都会注释掉该行，因此X不会读取它。通过注释而不是删除“ exec gnome-session”行，您可以稍后返回，取消注释，在“ exec dwm”上添加注释，然后继续使用Gnome。

窗口布局
如果在安装和启动中一切正常，则您应该盯着（主要是）空白屏幕。Dwm从其显示屏顶部的工具栏开始。

它使用三种布局-Tiled，Monocle和Floating-来显示窗口。要以这些布局打开程序，您需要学习一些命令。

使用dmenu
默认情况下，dwm使用dmenu打开系统上可用的程序。打开dmenu并搜索新程序的键盘命令是Mod1+ p，通常会转换为左Alt+ p键。

Dmenu将在屏幕顶部显示一个工具栏。您只需键入程序的名称（忽略大写字母，如“ firefox”），找到后按Enter。

平铺布局
打开三个程序，例如Firefox，feh和一个文本编辑器，您最终可能会看到如下图所示的几何图形。
这是dwm的默认模式：平铺。如果您处于其他任何模式，请按Mod1+ t到达此处。

此模式将立即显示您打开的所有程序。在上面的屏幕截图中，您可以看到Firefox位于左侧的“ Master”区域，而文本编辑器位于右侧的“ Stacking”区域。


除非进行更改，否则平铺模式会将一个程序置于“主”中，将其余程序置于“堆栈”中。在同一画面上可以与移动更多地砖为“Master” Mod1+ i，并减少他们Mod1+ d。在“主”区域中添加一些空间，并再打开一些窗口，如下图所示。
当您在工具栏中的数字1-9后面打印“ [] =”时，您就会知道自己处于平铺模式。

通过使用Mod1+ j向前移动和Mod1+ k向后移动，可以将焦点更改为该区域中的任何窗口。您也可以通过将鼠标移到窗口上方来选择一个窗口。按下Mod1+ Enter会将聚焦的窗口移到“主”区域。

单片布局
Monocle布局消除了“主”和“堆叠”区域。相反，它以全屏方式显示每个窗口。其余窗口位于焦点后面的窗口后面。

您可以使用Mod1+ 切换到Monocle布局m。这将在工具栏中将标记更改为“ [3]”。徽章中的数字将随您打开的窗口数而变化。
此屏幕截图显示了与第一个Tiled屏幕截图相同数量的程序，但是仅显示了活动窗口。用Mod1+ j和Mod1+ 切换活动窗口k。

浮动布局
当窗口可以在屏幕上浮动时，某些程序会更好地工作。使用Mod1+ 切换到浮动模式，f并看到标记更改为“> <>”。

这张照片显示了浮动模式下的Gimp。
使用鼠标在此模式下，您可以移动和调整窗口大小。按住Mod1鼠标左键拖动窗口；使用鼠标右键调整窗口大小。

杂项命令
记录以下其他命令以帮助您进行导航：

打开第一个终端：Mod1+ Shift+Enter
将视图切换到新的虚拟桌面：Mod1+[number 1-9]
将活动窗口移至新的虚拟桌面：Mod1+ Shift+[number 1-9]
退出dwm：Mod1+ Shift+q

阅读终端中的dwm手册页，以man dwm获取有关这些命令和其他命令的帮助。

结论
乍看起来，这似乎很费力，但是一旦掌握了dwm，它就很容易使用。

不要Mod1+ Shift+ q太快。当您完成日常任务时，您现在所做的工作将获得简单性，可预测性和速度的回报。
